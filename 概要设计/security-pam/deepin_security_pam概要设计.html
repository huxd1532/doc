    deepin-security-pam 实现了对 sudo, su 的限制，未打开开发者模式时禁止使用。 因此在设计时需要考虑系统的稳定性、架构兼容和扩展性。

  3. 术语说明

    • sudo : linux 下一个提权的命令，允许的用户以超级用户或其他用户的身份执行命令
    • su : linux 切换用户的命令，允许以指定的用户运行命令

1.0.2 系统设计

deepin-security-pam 主要是限制 sudo 和 su 命令带来的提权操作，而用户认证又是通过 PAM 完成，因此只需实现一个 PAM 模块来拒绝非开发者模式下的 sudo 和 su 的认证请求。

deepin-security-pam 的整体结构如下图：

待补

  1. 模块结构
    1. 提权限制模块

      提权限制模块实现了在非开发者模式下拒绝认证请求的功能，是一个 PAM 模块，因此需要按照 PAM 的规则实现以下接口：

      接口 描述
      pam_sm_authenticate 认证管理接口, 用于认证用户
      pam_sm_setcred 证书设置接口
      pam_sm_acct_mgmt 帐号管理接口, 用于修改账户信息
      pam_sm_open_session 打开会话
      pam_sm_close_session 关闭会话
      pam_sm_chauthtok 口令管理接口, 用于设置口令
  2. 关键流程设计

    deepin-security-pam 的功能很简单，其中的主要流程如下：

    1. 读取配置文件，判断是否处于开发者模式
    2. 如果未处于开发者模式，则返回 PAM_ABORT 取消此次认证流程
    3. 如果处于开发者模式，则返回 PAM_IGNORE 继续下面的 PAM 流程
  3. 关键数据结构设计

    deepin-security-pam 核心就是实现 PAM 指定的接口，定义如下： 

    struct pam_module {
    const char *name;       /* Name of the module */

    /* These are function pointers to the module's key functions.  */

    int (*pam_sm_authenticate)(pam_handle_t *pamh, int flags,
                   int argc, const char **argv);
    int (*pam_sm_setcred)(pam_handle_t *pamh, int flags,
              int argc, const char **argv);
    int (*pam_sm_acct_mgmt)(pam_handle_t *pamh, int flags,
                int argc, const char **argv);
    int (*pam_sm_open_session)(pam_handle_t *pamh, int flags,
                   int argc, const char **argv);
    int (*pam_sm_close_session)(pam_handle_t *pamh, int flags,
                int argc, const char **argv);
    int (*pam_sm_chauthtok)(pam_handle_t *pamh, int flags,
                int argc, const char **argv);
};

    这个结构体中定义不同的 PAM 操作需要的接口， deepin-security-pam 需要按需实现。

1.0.4 部署与实施

deepin-security-pam 部署比较简单，使用 make 构建完成后安装即可。

  • 编译： make
  • 安装：因为需要限制 sudo 和 su 命令，因此需要需要对应的 pam 文件

    • sudo

      需要修改 /etc/pam.d/sudo 文件，将 auth requisite deepin_security_verify.so 添加到文件的首行

    • su

      需要修改 /etc/pam.d/su 文件，将 auth requisite deepin_security_verify.so 添加到文件的首行
